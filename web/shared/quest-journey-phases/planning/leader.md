# Planning -- Leader Brief

Use this phase after the user has already approved the initial Journey and scheduling plan.

Leader actions:
- Keep the board row in `PLANNING`.
- Send the standard planning-only dispatch.
- Require the worker to load the quest skill, read and claim the quest, return a plan, then stop.
- Review the returned plan yourself first. If it stays within the approved Journey and introduces no significant ambiguity, scope change, Journey revision, user-visible tradeoff, or other blocking issue, approve it yourself and dispatch the next phase.
- Escalate back to the user only when the plan introduces one of those issues or otherwise genuinely needs user approval.
- If the worker recommends a Journey revision, decide whether to revise the board before advancing and whether that revision needs fresh user approval.
