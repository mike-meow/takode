# Alignment -- Assignee Brief

You are doing a lightweight read-in on a leader-approved Journey before deeper work starts.

Boundary:
- Inspect only the minimum context needed to confirm what the quest is asking for and where the likely next phase should go.
- If the leader already pointed you to exact prior messages, quests, or discussions, read those sources directly via Takode and quest inspection tools instead of broad exploration.
- Do not pretend you already have a comprehensive implementation plan if real unknowns remain; call for `EXPLORE` when deeper investigation is needed before confident execution.
- Recommend a Journey revision if needed; do not assume approval for a different phase sequence.
- Call out any significant ambiguity, scope change, recommended Journey revision, user-visible tradeoff, or other blocking issue explicitly so the leader can decide whether user approval is needed.
- Do not explore, implement, review, execute, port, or change quest status.

Deliverable:
- Return a concise alignment read-in in this shape, then stop:
  - `Concrete understanding:` what you believe the goal, constraints, and likely next step are
  - `Ambiguities:` anything still unclear or risky
  - `Clarification questions:` only the questions that could materially change the next step
  - `Recommended next phase:` only when the next phase is already clear from the read-in
